What is native comp / GCC emacs ?

  • Gcc emacs/ emacs native comp/ emacs 28 (have native comp option) are all names for the same thing. Emacs wherein all of the lisp files are compiled down to C code using just in time compilation.
  • this fixes my main qualm about getting deep into the emacs ecosystem and kept me with one foot in neovim…
    • Elisp is old, kindof slow, and the code base is stagnating
    • this makes it a hard sell for many people on the fence like myself. This was the main draw of neovim for me. I love that they went with lua and it seemed almost like an emacs inspired version of vim, faster than both, and almost as extensible as emacs with the convenience of vim.
  • With JIT compilation everything (in theory) should be faster. I used to get crashes when dealing with large files but have since not experienced any. Finding files, opening & changing buffers all feels crisper and sometimes I forget I am not in a terminal or using nvim and opening / closing buffers. It might be placebo but I also feel like lsp mode is snappier and I get instant completions with every keystroke.
  • The last green flag for me is that gcc emacs is available via the nix package manager. That means that I have the same emacs on my linux / mac os machines. Although there are customized Mac OS specific emacs versions in Brew and Nix I prefer having the same version on all machines so I know what to expect and can have the same config across machines without worrying about the occasional incompatibility.

Getting it installed (via nix)

  • I find nix extremely fascinating, the idea of a functional package-manager that can make your system much more resistant to breaking and instability and works cross platform is one linux users can rally behind. What’s even better is that it works on mac os. This means I can use the same packages across machines. I want to learn about package management accessories if you will like darwin and homemanager but for know I am sticking with nix-env -iA nixpkgs.[packagename].
  • installing gccEmacs in this manor is extremely simple it first requires adding the correct link to an emacs.nix file and then you’re all set!
mkdir ~/.config/nixpkgs/overlays/ # if these folders don't exist you have to make them
cd  ~/.config/nixpkgs/overlays/
echo "import (builtins.fetchTarball {
      url = https://github.com/nix-community/emacs-overlay/archive/master.tar.gz;
    })" >> emacs.nix
cd
nix-env -iA nixpkgs.emacsGcc # now just wait ~30 minutes depending on your system
